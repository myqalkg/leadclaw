# Lead-Claw 设计部输出技能标准 (v1.2)

## 📌 核心准则：拒绝自嗨，锁定小程序
1. **画布强制规范**: 必须锁定在 `375px` (iPhone 标准) 画布中心输出。
2. **容器约束**: 外部容器必须具备 `box-shadow` 和 `border-radius: 40px` 以模拟手机壳体感。
3. **安全区适配**: 底部必须通过 `env(safe-area-inset-bottom)` 预留 iPhone 交互条空间。

## 🎬 交互红线 (必须可点)
- **模拟弹窗**: 必须通过 JavaScript 实现微信原生的上滑弹窗 (Modal) 反馈。
- **触感反馈**: 所有卡片和按钮必须具备 `active:scale` 或 `opacity` 的点击态。
- **业务耦合**: 点击商品卡片必须触发“API 验证中”或“正在转链”的弹窗提醒，辅助研发理解逻辑。

## 🎨 视觉锚点 (V5.5 唯一真相)
- **Header**: 包含“胶囊避让”和“数据实时同步”状态。
- **Auth-Bar**: 紧贴 TabBar 的黑色半透明授权条。
- **Color**: 品牌橙 `#EA580C`, 警告红 `#FF4D4F`, 背景灰 `#F7F7F7`。

---
**执行准则**: 禁止任何形式的 Web 全屏化设计，禁止静态死图输出。
