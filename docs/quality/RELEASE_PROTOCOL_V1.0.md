# Lead-Claw (领爪) 标准化研发上线流程协议 (V1.0)

## 📌 核心准则
本项目所有功能发布必须遵循「研-测-审-运」四位一体流程，严禁跳过任何环节。

## 🛠️ 第一阶段：研发施工 (R&D)
- **标准输入**: 设计部签发的 `SKILL_DESIGN.md`。
- **输出要求**: 小程序原生 WXML/WXSS/JS 代码包。
- **自检**: 必须自查 `app.json` 基础配置及环境路径。

## 🛡️ 第二阶段：质量验证 (QA)
- **执行人**: 质量部审计员。
- **检查项**: 
    - [ ] 设计还原度 (375pt 画布适配)。
    - [ ] 核心流程阻断性 (授权弹窗逻辑)。
    - [ ] 真实 API 数据稳定性 (蜜源接口)。
- **输出**: `QA_VALIDATION_REPORT.json` (状态必须为 `CHECK_PASSED`)。

## 🎨 第三阶段：产设联审 (PM/UE)
- **执行人**: 产品经理与设计负责人。
- **重点**: 真机交互体感走查，确认底部 TabBar 与安全区适配。
- **批准**: 签署 `READY_FOR_PROD` 文档。

## 🚀 第四阶段：运维发布 (Ops)
- **执行人**: 运维组。
- **动作**: 小程序后台代码上传、预览二维码分发、上线。

---
**执行标准**: 任何未经 QA 验证或产设确认的发布将被视为“生产事故”，需直接追责并回滚。
