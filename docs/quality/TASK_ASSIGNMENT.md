# Lead-Claw (领爪) 质量管理组 - 任务书 (v1.0)

## 1. 核心使命
确保平台输出的每一条“神价”不仅在代码层面调通，更在业务逻辑、数据准确性和推送体验上达到“商用级”零差错。

## 2. 第一阶段 (Sprint 2) 重点任务

### 2.1 数据真实性校验 (Data Integrity)
- **任务**: 编写监控脚本，交叉比对 `gzmiyuan.com` 接口返回的 `voucherPrice` 与电商原链真实价格是否一致。
- **红线**: 严禁出现“虚假降价”或“券已失效”的消息推送至用户侧。

### 2.2 推送链路压力测试 (Load Testing)
- **任务**: 模拟瞬时 5w+ 级别的流量突发（多个“神价格”同时出现时）。
- **指标**: 微信 Push 接口调用成功率 > 99.9%，延迟控制在 3s 以内。

### 2.3 极致兼容性测试 (Compatibility)
- **任务**: 针对 V5.5 页面，进行不同机型（尤其是一屏显示高度不同的手机）的覆盖测试。
- **目标**: 确保底部 TabBar 和“授权引导浮窗”在低分辨率屏幕下不重叠。

### 2.4 CPS 收益闭环审计 (Revenue Audit)
- **任务**: 验证【领爪】内部转链逻辑。
- **验证点**: 确保 `goodsId` 在二转链接后，能正确携带平台指定的 PID，防止佣金流失。

## 3. 每日巡检要求
- 每天 10:00 前提交一份【全网健康度日报】至总负责人可视化看板。
- 流水线发布前，须由 QA 签发【准入证书】。
